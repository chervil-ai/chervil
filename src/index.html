Provider & model — bring your own API

Switch providers to save money, work offline, or use a model you prefer. Each has trade-offs — pick what fits the task. Claude, Grok, Gemini, and OpenAI do live web grounding (real, cited sources); Azure and Ollama compose from the model's own knowledge. Claude also offers the full two-phase Deep Dive.

Model

Audio narration (text-to-speech)

Used by the 🔊 Audio button on composed pages. Voices come from your operating system — look for ones labelled "Natural" or "Online" for the best quality.

Voice input (speech-to-text)

Talk to Sprig with the 🎤 button in the chat box. Recordings are sent to a Whisper-compatible transcription service of your choice (OpenAI, Groq, Azure, or a local whisper server) — Claude can't transcribe audio, so this uses its own key. The audio is sent only to this endpoint, never stored.

Publishing (Chervil Pro)

Publish a lesson, quiz, or any interactive page (clock, calculator, converter…) to a shareable getchervil.com link. Sign in at getchervil.com/me, generate a publish token, and paste it here. Then use 🌐 Publish to web from any page's Export menu. (Pages that call Sprig at runtime won't be interactive when hosted.)

Sync between computers

Point Chervil's data (tabs, history, Spaces, agents) at a folder your cloud client already syncs — inside OneDrive, Google Drive, or Dropbox. Set the same folder on each computer and your sessions follow you. Note: API keys aren't synced (they're encrypted per-machine — re-enter once per computer), and running Chervil on two machines at once is last-writer-wins.

Appearance

Choose how tabs are arranged. Either way, you can click-hold-drag a tab to reorder it, just like a web browser.

Notifications

Living pages (the ● live control on a composed page) auto-refresh on a schedule. Chervil keeps refreshing them even when minimized or in the background.

Listening — “Hey Sprig”

Let Chervil listen for a wake word and pop the Quick-Ask bar so you can speak a request hands-free. Detection runs fully on-device and free (openWakeWord, no account or key) — your mic audio never leaves the machine until you actually speak a command, which is then transcribed by your configured voice-input service above.

Wake word Built-in words work immediately. For the literal “Hey Sprig”, train a free openWakeWord model (a single-command trainer or the openWakeWord Colab) and load the resulting .onnx here.

MCP servers Claude only

Connect remote Model Context Protocol servers to give Sprig extra tools and data (e.g. your calendar, a database, a SaaS app). Sprig can call their tools while composing a page. Their tools run on Anthropic's side as part of the request — so only add servers you trust, and prefer read-only ones. Works with the Claude provider only; other providers ignore these.

Remote URL servers only (SSE/HTTP). Local/stdio servers aren't supported yet. The token is stored on this machine and sent to Anthropic as the server's authorization.

Images

Claude can't generate images, so Chervil uses your OpenAI, Google Gemini, or Grok (xAI) key to make a hero image. Off by default — image generation costs money (billed to your own key). One image per composed page.

Passwords encrypted

Save logins for real sites and fill them back in with the 🔑 button. Passwords are encrypted with a master passphrase (and your OS keychain) and never shown to Sprig, never sent anywhere. Sprig itself never types passwords. The vault auto-locks (see below) and asks for your passphrase again before filling. (Windows Hello unlock is planned.)

Autofill

Save your details to fill forms on real sites — on a website, just say “fill the form.” Passwords and card details are never stored or auto-filled.

About you (personal memory)

Tell Sprig about yourself — location, interests, work, preferences, how you like answers. Sprig uses this to tailor every page it composes (kept on this machine, sent only to your chosen model).
